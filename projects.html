Tesla's Revenge

Active Work Dates : Fall 2015 - Spring 2017
Role(s): Programmer
Languages Used: C# (Unity Engine)

Project Description:

  My first experience with programming in Unity. I joined the project about a 6 months into its development and began to work on small sections like character movement. At the time, the team was having problems with the player character phasing through the walls and other objects with 2D BoxCollider components. I was able to devise a system of raycasts that prevented this problem from occuring.

  Immediately after completing this problem, I was given the ability to make a minigame that would go into the main project. This minigame took about two to three weeks to go from idea to actualized game. After helping with some other miscellaneous parts of the project, I was again assigned another minigame to work on. It was more complicated than the last one, but it took only slightly longer to complete, about 3 weeks. I must not that in both of these minigames I only worked on the programming and design aspects. I was aided in the art department by another member of the team.

Project Technical Components

  • Player movement controls
  • Multiple minigames (each with a unique system)
  • NPC interaction
  • NPC dialog trees

Ping

Active Work Dates : Summer 2017
Role(s): Game Designer, Programmer
Languages Used: C# (Unity Engine)

  A small personal project aimed at refamiliarizing myself with Unity for an upcoming class at the time. The concept of the game is that the player has to navigate a maze and find the exit. A normally simple task, but unfortunately the player is 'blind' and must find the way using sound pings (a way to see what is in the immediate area around them). The project was intended for a mobile device though the appropriate touch controls were not implemented.

Project Technical Components

  • JSON-based progress saving
  • Player movement controls
  • Local area object highlighting
  • Hazards
  • Player death / respawn
