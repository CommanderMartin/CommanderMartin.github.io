Western Wildlands: Journey to Oregon

Active Work Dates : Spring 2018
Role(s): Game Designer, Programmer, Team Lead
Languages Used: C# (Unity Engine)
Platform: PC

 

Project Technical Components

  • Player and container inventory system
  • Wagon, NPC interaction systems
  • NPC shop system
  • Wagon follow-path system
  • JSON-based progress saving and loading
  • Player movement controls

Project Information:

  This project came out of the class immediately after the one that Lost Souls' Arcade was created in. Instead of taking place in a confined area, this one was set to be semi-open world where the player would advance through different levels as they completed objectives. The game was intended to be a sort of '3D Oregon Trail' where the player would go on a journey with 3 other NPCs and a wagon.

  The player set as the party leader and with the job to get not only themselves but also as many of their NPCs as possible to the end of the journey. The game was planned to have 11 levels, each one harder than the previous. The player would be incentivised to prepare for the later stages as early as possible. The game is currently unfinished with only 2 of the 11 stages complete, and only some core mechanics with partial implementation.

Languages Used: C# (Unity Engine)
Platform: Mobile

 

Project Technical Components

  • JSON-based progress saving and loading
  • Local area object highlighting
  • Player movement controls
  • Hazards

Project Information:

  A small personal project aimed at refamiliarizing myself with Unity for an upcoming class at the time. The concept of the game is that the player has to navigate a maze and find the exit.
